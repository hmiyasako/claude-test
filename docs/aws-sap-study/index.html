Well-Architected Framework

AWS設計のベストプラクティス

AWSの設計原則を体系化した6つの柱。SAP試験の全ての問題はこのフレームワークに基づいて設計されています。

6つの柱 設計原則
🔄

疎結合アーキテクチャ

Loose Coupling

コンポーネント間の依存関係を最小限にし、障害の影響範囲を限定。SQS、SNS、EventBridgeを活用した非同期設計。

非同期処理 障害分離
📊

スケーラビリティ設計

Scalability Patterns

水平スケーリングを優先。ステートレス設計、Auto Scaling、ElastiCache、CloudFrontでスケーラブルなシステムを構築。

水平スケール ステートレス
🎯

サーバーレスファースト

Serverless First

運用負荷を最小化するためにLambda、API Gateway、DynamoDB、Step Functionsなどマネージドサービスを優先的に選択。

運用負荷軽減 従量課金
🏢

AWS Organizations

マルチアカウント管理

複数AWSアカウントを一元管理。OU構造、SCP（Service Control Policy）によるガバナンス、一括請求でコスト管理。

OU構造 SCP
🗼

Control Tower

ランディングゾーン構築

マルチアカウント環境のベストプラクティスを自動構築。ガードレール（予防的・検出的）でコンプライアンスを強制。

ガードレール 自動プロビジョニング
🔐

Service Control Policy

SCPによるアクセス制御

Organizations全体でIAMポリシーの上限を設定。特定リージョン禁止、特定サービス禁止などの組織レベル制御。

境界設定 ガバナンス
🏠

マルチアカウント戦略

Account Vending Machine

環境別（dev/stg/prod）、ワークロード別、チーム別のアカウント分離。セキュリティ境界の明確化とブラスト半径の縮小。

環境分離 ブラスト半径
🔗

AWS RAM

Resource Access Manager

アカウント間でリソースを共有。Transit Gateway、サブネット、License Manager設定などを組織内で効率的に共有。

リソース共有 コスト削減
🌐

VPC設計パターン

マルチティア・マルチAZ

パブリック/プライベートサブネット分離、3AZ構成、NATゲートウェイの冗長化。CIDR設計でIP重複を回避。

サブネット設計 CIDR計画
🔀

Transit Gateway

ハブ＆スポーク接続

複数VPCやオンプレミスをハブで接続。ルートテーブルでトラフィック制御、マルチリージョンピアリング対応。

ハブ＆スポーク ルート伝播
🔌

Direct Connect

専用線接続

オンプレミスとAWSを専用線で接続。LAG構成で冗長化、パブリック/プライベートVIF、Transit VIFの使い分け。

専用線 低レイテンシー
🔒

PrivateLink

プライベート接続

VPCエンドポイントでAWSサービスにプライベートアクセス。インターフェース型とゲートウェイ型の使い分け。

VPCエンドポイント インターネット不要
🛡️

ネットワークセキュリティ

多層防御

Security Group（ステートフル）、NACL（ステートレス）、WAF、Shield、Network Firewallの組み合わせ。

多層防御 DDoS対策
🌍

Route 53

DNSとトラフィック管理

ルーティングポリシー（シンプル、加重、レイテンシー、フェイルオーバー、位置情報）でグローバルトラフィック制御。

DNS ヘルスチェック
🚀

6つのR（移行戦略）

Migration Strategies

Rehost、Replatform、Refactor、Repurchase、Retain、Retire。ワークロードに応じた最適な戦略を選択。

リフト＆シフト モダナイゼーション
📦

Migration Hub

移行の一元管理

移行プロジェクトの進捗を一元管理。Application Discovery Service、Server Migration Service、DMSと連携。

進捗管理 ダッシュボード
🗄️

Database Migration Service

DBの移行・レプリケーション

同種・異種DB間の移行。SCT（Schema Conversion Tool）でスキーマ変換、CDC（継続的変更キャプチャ）対応。

異種DB変換 CDC
❄️

Snow Family

大容量データ転送

Snowcone（8TB）、Snowball Edge（80TB）、Snowmobile（100PB）。帯域制限やセキュリティ要件に対応。

オフライン転送 エッジコンピューティング
🔄

Application Migration Service

MGN（リフト＆シフト）

サーバーのブロックレベルレプリケーション。カットオーバーまで継続的同期、最小ダウンタイムで移行。

継続的レプリケーション 最小ダウンタイム
👤

IAM設計

最小権限の原則

ロールベースアクセス、IAM Identity Center（旧SSO）でフェデレーション、Permission Boundaryで権限上限設定。

最小権限 フェデレーション
🔑

KMS / CloudHSM

暗号化キー管理

KMSでマネージドキー管理、CloudHSMで専用ハードウェア。キーポリシー、グラント、自動ローテーション設定。

CMK キーローテーション
🔍

Security Hub

セキュリティ状況の一元管理

GuardDuty、Inspector、Macie、Firewall Managerからの検出結果を集約。AWS基礎セキュリティベストプラクティス適用。

統合ダッシュボード コンプライアンス
🕵️

GuardDuty

脅威検出

VPC Flow Logs、CloudTrail、DNS Logsを分析して脅威を検出。機械学習ベースの異常検知、Organizations統合。

脅威検出 ML分析
🔏

Secrets Manager

シークレット管理

DBパスワード、APIキーの安全な保存と自動ローテーション。Lambda連携でカスタムローテーション、クロスアカウントアクセス。

自動ローテーション 監査
📋

AWS Config

設定変更の追跡

リソース設定の変更履歴を記録。Configルールでコンプライアンスチェック、自動修復アクション設定。

構成管理 自動修復
💰

Reserved Instances

予約購入による割引

1年/3年のコミットメントで最大72%割引。Standard RIとConvertible RI、容量予約の有無を選択。

長期コミット 最大72%割引
💵

Savings Plans

柔軟なコミットメント

1時間あたりの使用量をコミット。Compute Savings Plans（EC2、Fargate、Lambda対応）とEC2 Instance Savings Plans。

柔軟性 サービス横断
🎯

Spot Instances

最大90%割引

余剰キャパシティを利用。中断に強いワークロード向け。Spot Fleet、EC2 Auto Scaling混合インスタンスポリシー。

バッチ処理 中断許容
📈

Cost Explorer

コスト分析・予測

コストと使用量の可視化、予測、RI/Savings Plans推奨事項。コスト配分タグで部門別コスト追跡。

可視化 予測
⚠️

Budgets

予算アラート

コスト、使用量、RI/Savings Plans利用率の予算設定。SNS通知、自動アクション（EC2/RDS停止）でコスト超過防止。

アラート 自動アクション
🏷️

タグ戦略

コスト配分とガバナンス

必須タグ（Environment、Owner、CostCenter）の強制。Tag Policies、AWS ConfigでタグコンプライアンスをチェックV

コスト配分 リソース管理
🔄

DR戦略

ディザスタリカバリパターン

Backup & Restore、Pilot Light、Warm Standby、Multi-Site Active-Active。RPO/RTOに応じた戦略選択。

RPO/RTO 4つの戦略
🌏

マルチリージョン設計

グローバルレジリエンス

Aurora Global Database、DynamoDB Global Tables、S3 Cross-Region Replication、Global Acceleratorの活用。

クロスリージョン データ同期
📊

Auto Scaling

自動スケーリング

ターゲット追跡、ステップスケーリング、スケジュールスケーリング。予測スケーリングでプロアクティブに対応。

自動化 予測スケーリング
💾

AWS Backup

バックアップの一元管理

EC2、EBS、RDS、DynamoDB、EFS、FSxのバックアップを一元管理。クロスリージョン・クロスアカウントコピー。

一元管理 ポリシーベース
🔧

Resilience Hub

レジリエンス評価

アプリケーションのレジリエンス目標（RTO/RPO）を定義し、現在のアーキテクチャを評価。改善推奨事項を提示。

評価 推奨事項
🧪

Fault Injection Simulator

カオスエンジニアリング

制御された障害を注入してシステムの回復力をテスト。EC2停止、レイテンシー追加、AZ障害シミュレーション。

カオステスト 障害注入
